トレンド比較編

開発言語

機械学習関連でよく用いられるプログラミング言語はPython、MATLAB及びR言語がある。

Python

Pythonのシェアは他の言語と10倍近い差をつけてトップとなっている。

他の言語に比べて複雑な記述を必要とするが、高度な分析が可能であるため、開発言語としてはPythonを使用されることが多い。

MATLAB

数学的処理を行うためのプログラミング言語。

高機能であるものの有料（学生は無料）であるため、使用する人が限られる。

R言語

統計解析を行うためのプログラミング言語。

Pythonに比べて利用率は高くない。

Pythonのコードの形式

Pythonは他のプログラミング言語と同じように、テキスト形式で記述して実行する形式（.py）と、Jupyterノートブックというものに記述して実行する形式（.ipynb）がある。

ここではそれぞれスクリプト形式、ノートブック形式と呼ぶ。

それぞれに利点があるため、これらを使い分けることができるとよい。

スクリプト形式（.py）

コードの密度が高いため、長いコードを書いたときの一覧性がよい。

またスクリプト形式で書いたコードはモジュールとして別のファイルから読み込むことができるため、大規模なコードに適している。

また開発環境によっては、スクリプト形式のほうが細かなデバッグができることが多い。

単なるテキストファイルのため、LinuxなどのCUIからでもプログラムを作成することが可能。

ノートブック形式（.ipynb）

コードの間隔が広く、装飾的なコメントも入れられるため、視覚的に理解しやすい。

セルと呼ばれる複数のコード単位で実行するため、プログラムの全体が完成していなくても実行を開始することができる。

そのため簡易的なコードやテストの実行などに向いている。

Jupyter NotebookやGoogle Colab、VSCodeなどの環境で使用できる。

開発環境

一般的な環境としてはVisual Studio Code（VSCode）、Google Colab及びJupyter Notebookの3つが挙げられる。

Visual Studio Code（VSCode）

標準で非常に多くの機能を有し、拡張機能でさらに機能を追加することができる。

デバッグを支援する機能が充実しているため、世界的にも非常に高いシェアを誇っている。

迷ったらこの開発環境を使えば間違いない。

Google Colab

Googleの提供する強力なサーバを利用できる開発環境。

コードはノートブック形式に限られ、時間制限などもあるが、手軽に利用できる。

Jupyter Notebook

ノートブック形式のコードを書くための原始的な環境。

現在はVSCodeでもノートブック形式を扱えるため、わざわざこの環境をセットアップしてまで利用する必要はない。

機械学習フレームワーク

Pythonで使用できる機械学習のフレームワークで、主流なものとしてTensorFlow及びPyTorchが挙げられる。

両者ともよく使われているため、どちらを使用してもよい。

しかし公開されているコードはこのどちらかでしか書かれていないため、どちらも理解できるようになっておかなければならない。

TensorFlow

Googleが開発した機械学習のフレームワーク。

実際に使用する際は、TensorFlowのラッパーであるKerasを使うことが多い。

PyTorchに比べて簡単に実装ができるようになっている。

PyTorch

Facebook（現：Meta）が開発した機械学習のフレームワーク。

TensorFlowに比べてデバッグがしやすく、また機能の拡張を行いやすいため、研究者で使用している人が多い。

コード管理（Git）

プログラムコードを管理するための方法の一つとして、Gitを用いる方法がある。

「Gitとは何か」は各自で調べてほしいが、Gitを使用することで過去のコードを管理できる。

Gitに対応したホスティングサービスとしてはGitHubとGitLabがあるが、大差はない。

GitHub

GitLabよりも古くから存在しており、利用者が多い。

Microsoftが買収しており、VSCodeとの連携にも期待ができる。

GitLab

比較的新参のサービスで、GitHubにない機能もある。

ただ高度な開発をしないのであれば、わざわざGitLabを使う必要はない。
